# Support

## Security

Report vulnerabilities through [SECURITY.md](SECURITY.md). Do not disclose
security issues in public GitHub issues, pull requests, or discussions.

## Project Questions

Use the owning repository for questions about a specific EvalOps project.

## Maintainers

Internal org automation, workflow templates, service catalogs, and operating
contracts live in the private `evalops/.github-private` repository. Maintainers
with access should route that work there.

## Public Contact

For general contact, use the contact path published on the EvalOps website.
